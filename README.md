# Documents Repository

## Purpose
This repository contains all official project documentation related to the System Design Project.

## Contents Structure

```text
documents/
│
├── SRS/
├── SDS/
├── Meeting-Notes/
├── Pitch-Deck/
├── Log-Forms/
└── Diagrams/
```

## Included Documents
- Software Requirements Specification (SRS)
- Software Design Specification (SDS)
- Meeting Notes
- Pitch Deck
- Weekly Log Forms
- Source files for diagrams

## Tools Required
- Microsoft Word / Google Docs
- Draw.io
- Lucidchart
- PowerPoint / canva

## Branch Strategy
- `main` → Approved documents only
- `draft/<document-name>` → Working drafts
- `fix/<issue-number>` → Corrections and review fixes

## Contribution Guidelines
Please follow the rules mentioned in CONTRIBUTING.md before pushing changes.

## Team Workflow
1. Create a draft branch
2. Make changes
3. Commit changes
4. Push to GitHub
5. Create Pull Request
6. Get approval
7. Merge into `main`
